RightLayout 1.3 is a reliability, UX, and localization release.

What changed:
- Reworked the correction runtime around stricter host profiles, safer replacement decisions, and more conservative feedback attribution.
- Improved behavior in non-Accessibility editors by narrowing automatic replay, tightening manual scope, and reducing whole-line replacement failures.
- Added stronger capability-aware replacement paths for AX and blind editors, with safer commit handling and better fallback boundaries.
- Rebuilt the macOS UI into a calmer control-center layout with an overview pane, unified diagnostics workspace, denser history, and clearer support states.
- Patched interface translations across the updated settings, diagnostics, onboarding, update, and menu bar surfaces, including correct system-language routing for dynamically generated UI text.
- Cleaned up release/version handling so packaged builds, `swift run`, and app metadata stay aligned.

Operational notes:
- Runs locally on macOS 13.0 and later.
- Requires Accessibility permission for full live correction; manual and diagnostics behavior may be more limited in unsupported editors.
